The Basics

What is Prompt Engineering?

It is the art and science of structuring inputs to get the most accurate, reliable, and useful outputs from an LLM.

  • Not just "talking to a bot"
  • Setting clear operational boundaries
  • Providing necessary domain context
  • Specifying exact output requirements
  • Managing the model's reasoning path

🔍 The Garbage In, Garbage Out Rule

Vague prompts produce vague results. Specificity is the antidote to hallucination.

🎯 The Goal

To reduce the "search space" for the model so it focuses only on the relevant solution.

Common Pitfalls

Why Vague Prompts Produce Vague Results

❌ The Vague Prompt

"Tell me about CRISPR."

Result: A generic, high-school level summary that misses specific technical nuances or recent breakthroughs.

✅ The Structured Prompt

"Explain the mechanism of CRISPR-Cas9 base editing vs. prime editing for a genomic researcher. Focus on indel frequency and target constraints."

Result: A technical comparison with specific metrics and biological context.

"The model cannot read your mind. It only knows what you tell it and what it has been trained on."
The Framework

Structuring with Role, Task, Format, & Constraints

🎭 Role

Who is the model?
"Act as a senior bioinformatician."

📋 Task

What must it do?
"Analyze these gene expression values."

📄 Format

How should it look?
"Return a Markdown table."

🚫 Constraints

What are the limits?
"Only use genes with p < 0.05."

# Putting it all together Role: Senior Pharmacologist. Task: Summarize the ADME profile of this compound. Format: Bullet points with specific values for t1/2 and LogP. Constraints: Use only the provided PDF text. Do not hallucinate data.
Agentic Strategy

Plan → Approve → Execute → Iterate

The gold standard for multi-step agentic tasks in scientific research.

📝

Plan

The agent outlines the steps it will take.

🤝

Approve

You review the plan and provide feedback.

🚀

Execute

The agent performs the technical work.

🔄

Iterate

Review output and refine the prompt.

Why "Approve" is Mandatory

Scientific workflows are expensive in both tokens and time. Catching a logic error in the Plan phase saves $ and prevents bad data.

Technical Limits

Understanding the Context Window

The context window is the model's active working memory. Everything in it is "seen" at once.

  • What fills it? Prompt, chat history, attached files, tool outputs.
  • What does it cost? Higher usage = slower responses + higher $/token.
  • Hallucination Trigger: As the window nears capacity, models lose focus on the middle ("Lost in the Middle").

📏 Size Matters

Claude 3.5: 200k tokens (~150k words)
Gemini 1.5 Pro: 2M tokens (~1.5M words)

🧠 Context Drift

In long sessions, the model may forget the initial system instructions or constraints. Use /reset or summarize frequently.

Managing Long Sessions

🧹 Context Cleaning

Use /forget or clear chat history once a sub-task is complete to keep costs down and focus high.

📦 Modular Prompts

Instead of one massive prompt, break tasks into smaller modules. Use the output of one as the input for the next.

Hands-On

Hands-on: Prompt Engineering

Task: Literature Summarizer

  1. Open your agent.
  2. Provide a Role (e.g., "Act as a PI reviewing a new postdoc's abstract").
  3. Provide a Task (Summarize a scientific abstract you find on PubMed).
  4. Add Constraints (Max 100 words, highlight the methodology, mention one limitation).
  5. Apply Iterate: "Now rewrite this for a clinical audience."

Reflection

  • Did the Role change the tone?
  • Did it respect all Constraints?
  • How much did the session cost (check /stats)?

💡 Pro Tip

Save your best prompt structures in a PROMPTS.md file for reuse.

Summary

Session 02 Key Takeaways

  • Prompt Engineering = Structuring inputs for better outputs.
  • RTFC: Role, Task, Format, Constraints.
  • P-A-E-I ensures safety and logical flow.
  • Vague prompts = vague, unreliable results.
  • Context window = model working memory.
  • Manage your context to control costs and accuracy.
  • Approve plans BEFORE execution.
  • Iterate until the result is scientifically sound.
